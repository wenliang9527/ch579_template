# CH579 串口转以太网透传网关 - 使用说明

## 一、项目概述

本项目基于 WCH CH579M 微控制器，实现了一个动态分组路由的串口转以太网透传网关。

### 硬件平台
- 芯片: CH579M (ARM Cortex-M0)
- 时钟: 32MHz
- RAM: 32KB
- Flash: 256KB

### 核心功能
- 4路串口 + 1路以太网的数据透传
- 动态分组路由机制
- 20ms帧间隔控制
- 最大1024字节帧长度

---

## 二、分组路由机制

### 2.1 分组定义

| 分组 | 包含端口 | 说明 |
|------|---------|------|
| A组 | UART0, UART1, ETH | 主控通信组 |
| B组 | UART2, UART3 | 设备通信组 |

### 2.2 动态路由规则

```
A组任一端口收到数据 -> 标记A组活动端口 -> 转发给B组活动端口
B组任一端口收到数据 -> 标记B组活动端口 -> 转发给A组活动端口
```

### 2.3 初始状态
- A组默认活动端口: UART1
- B组默认活动端口: UART3

### 2.4 路由示例

场景1:
1. UART0收到数据 -> 标记A组=UART0 -> 转发给B组活动端口(UART3)
2. UART2收到数据 -> 标记B组=UART2 -> 转发给A组活动端口(UART0)

场景2:
1. TCP收到数据 -> 标记A组=ETH -> 转发给B组活动端口(UART3)
2. UART3收到数据 -> 标记B组=UART3 -> 转发给A组活动端口(ETH)

### 3.2 核心模块说明

#### data_forward.c - 数据转发模块

主要功能:
- 帧缓冲管理
- 动态路由决策
- 20ms发送间隔控制

关键函数:

| 函数 | 功能 |
|-----|------|
| DataForward_Init() | 初始化转发模块 |
| DataForward_RecvByte() | 接收字节存入缓冲区 |
| DataForward_FrameComplete() | 帧完成处理，加入发送队列 |
| DataForward_RecvFromEth() | 网口数据接收处理 |
| DataForward_Process() | 主循环调用，处理发送队列 |
| DataForward_UpdateActivePort() | 更新活动端口标记 |
| DataForward_GetPeerGroupMask() | 获取对方组活动端口掩码 |

全局变量:

```c
PortType_t g_groupA_ActivePort = PORT_UART1;  /* A组活动端口 */
PortType_t g_groupB_ActivePort = PORT_UART3;  /* B组活动端口 */
```

#### uart.c - 串口驱动

串口配置:

| 串口 | TX引脚 | RX引脚 | 分组 |
|-----|--------|--------|------|
| UART0 | PA14 | PA15 | A组 |
| UART1 | PA9 | PA8 | A组 |
| UART2 | PA7 | PA6 | B组 |
| UART3 | PB21 | PB20 | B组 |

中断处理流程:
```c
UARTx_IRQHandler() {
    case UART_II_RECV_RDY:      // 接收数据，存入缓冲区
    case UART_II_RECV_TOUT:     // 帧超时，完成帧接收
        DataForward_FrameComplete(port);  // 触发转发
}
```

#### main.c - 主程序

初始化流程:
```c
main() {
    1. 系统时钟初始化 (32MHz)
    2. 外设初始化
       - uart_init()      // 串口
       - net_init()       // 以太网
       - time0_init()     // 定时器
       - DataForward_Init() // 数据转发
    3. TCP服务器初始化 (端口6666)
    4. 主循环
       - CH57xNET_MainTask()      // 网络任务
       - CH57xNET_HandleGlobalInt() // 中断处理
       - DataForward_Process()     // 数据转发
}
```

## 六、使用说明

### 6.1 编译烧录

1. 使用 Keil uVision 5 打开 project/project.uvprojx
2. 选择目标芯片 CH579M
3. 编译 (F7)
4. 使用 J-Link 或 WCH-Link 烧录

### 6.2 连接方式

A组连接 (UART0/UART1/ETH):
- 连接上位机或主控设备
- 默认活动端口: UART1

B组连接 (UART2/UART3):
- 连接下位设备
- 默认活动端口: UART3

### 6.3 通信流程

1. A组发送 -> B组接收:
   - 通过UART0/UART1/ETH发送数据
   - 数据自动转发给B组当前活动端口

2. B组发送 -> A组接收:
   - 通过UART2/UART3发送数据
   - 数据自动转发给A组当前活动端口

3. 活动端口切换:
   - 哪个端口发送数据，就成为该组的活动端口
   - 后续数据将转发给新标记的活动端口

## 七、注意事项

1. 帧长度限制: 最大1024字节，超过会丢弃
2. 帧间隔: 最小20ms，防止数据拥塞
3. 队列深度: 3帧，满时新帧会被丢弃
4. 活动端口: 重启后恢复默认值(UART1/UART3)
